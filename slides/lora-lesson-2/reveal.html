② PEFT 家族 与 QLoRA 原理

15 min · 把"为什么 4bit 还能训"讲清楚

2.1 PEFT 家族全景

方法核心思想代表
Prompt Tuning学一段可训练的 soft promptGPT-3 soft prompt
Prefix Tuning每层注入可训练的 prefix KVLi & Liang 2021
LoRA权重更新低秩分解 $W + BA$Hu et al. 2021
QLoRA4bit 量化底座 + LoRADettmers 2023
DoRA / AdaLoRA幅度/方向解耦、自适应秩2024+

工业界主流：LoRA / QLoRA（简单、快、adapter 小）

2.2 QLoRA 一页纸

原模型 Qwen2.5-1.5B (fp16, 3GB) ↓ 4bit NF4 量化 量化底座 (0.9GB) ────冻结不训练──── + 每个 Linear 外挂 LoRA (A, B) ── fp16 训练 ── ↓ 前向：h = dequant(W_4bit) @ x + (α/r) · B·A·x 反向：只对 A, B 求梯度

三项关键技术：NF4 量化 · 双重量化 · Paged Optimizer

效果：1.5B 模型在 Kaggle T4 (16GB) 上训练占用 ~6GB

2.3 为什么能压到这么小？

存储项全量微调QLoRA
模型参数3 GB (fp16)0.9 GB (4bit)
梯度3 GB~35 MB
AdamW state6 GB (2×)~70 MB
激活 (MAX_LEN=512, bs=2)~4 GB~4 GB
合计~16 GB~5 GB
🎯 Kaggle T4 (16GB) 全量微调 1.5B 都已经很勉强，QLoRA 可以轻松跑 7B

④ peft 三板斧

75 min · 训练 + 合并 + 推理 + 评测

4.1 peft 三板斧

① LoraConfig（怎么挂）


LoraConfig(
    r=8, lora_alpha=16, lora_dropout=0.05,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],
    task_type="CAUSAL_LM", bias="none",
)

② get_peft_model（挂上去）


model = get_peft_model(base_model, config)
model.print_trainable_parameters()
# trainable: 8.8M / 1.55B (0.566%)

4.2 peft 三板斧（续）

③ save_pretrained / merge_and_unload


# 只存 adapter（30-40 MB）
model.save_pretrained("./adapter")

# 或：合并回底座，导出完整模型（需非量化底座）
merged = PeftModel.from_pretrained(base_fp16, "./adapter").merge_and_unload()
merged.save_pretrained("./merged")

两种部署形态：

  • Adapter only：热切换多任务，推理多一次矩阵乘
  • Merged model：部署简单，但失去热切换能力

4.3 训练配置（常用）


TrainingArguments(
    num_train_epochs=3,
    per_device_train_batch_size=2,
    gradient_accumulation_steps=4,     # 有效 bs=8
    learning_rate=2e-4,                # LoRA 专用，比全量 FT 大 10×
    warmup_ratio=0.05,
    fp16=True,
    gradient_checkpointing=True,        # QLoRA 必开
    optim="paged_adamw_8bit",           # QLoRA 专属
)

Kaggle T4 训练时间：3-5 分钟 / 3 epochs
