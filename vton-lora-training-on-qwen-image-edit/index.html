👔 基于 Qwen-Image-Edit 的虚拟试衣 LoRA 训练：从零到一的完整规划

最近在做虚拟试衣（Virtual Try-On）项目迁移时，做了一个重要决策：底座从 FLUX-dev.1 切到 Qwen-Image-Edit。原因很简单——FLUX-dev.1 是非商用协议，论文能发但产品上不了线。而 Qwen-Image-Edit 全系列都是 Apache-2.0，一条路走到底：学术产出和商业交付可以共振。

于是我花了两周时间研究怎么在 Qwen-Image-Edit 上训练 VTON LoRA。踩了不少坑，也参考了社区里已经跑通的案例。这篇文章把整个规划过程系统地记录下来，作为后续实际动手的路线图。

💡 如果你还不了解 Qwen-Image-Edit 是什么，建议先读我的另一篇文章：Qwen-Image-Edit 完全指南。如果你对传统 VTON 架构感兴趣，可以看 JCo-MVTON 完全指南。

为什么选 Qwen-Image-Edit 做 VTON？

先说结论——传统 VTON 方案（IDM-VTON、CatVTON 等）和基于 Qwen-Image-Edit 的 LoRA 方案，解决问题的思路完全不同：

┌────────────────────────────────────────────────────────────────┐
│             传统 VTON vs LoRA-based VTON                        │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│  传统方案 (IDM-VTON / CatVTON / JCo-MVTON):                    │
│  ┌──────────┐   ┌─────────────┐   ┌──────────┐   ┌─────────┐  │
│  │ 衣服图片 │ → │ 几何变形模块 │ → │ 语义分割  │ → │ 合成输出 │  │
│  │ 人体图片 │ → │ (TPS/Flow)  │ → │ (Parsing) │ → │         │  │
│  │ 姿态估计 │ → │             │ → │           │ → │         │  │
│  └──────────┘   └─────────────┘   └──────────┘   └─────────┘  │
│  特点：需要完整 pipeline，每步单独训练，结构复杂                   │
│                                                                 │
│  LoRA 方案 (Qwen-Image-Edit + LoRA):                           │
│  ┌──────────┐   ┌─────────────────────────────┐   ┌─────────┐  │
│  │ 衣服图片 │ → │                              │ → │ 合成输出 │  │
│  │ 人体图片 │ → │   预训练 Edit 模型 + LoRA    │ → │         │  │
│  │ 文字指令 │ → │   (端到端，一步到位)         │ → │         │  │
│  └──────────┘   └─────────────────────────────┘   └─────────┘  │
│  特点：利用基座模型的强大理解力，用 LoRA 教它"换装"这个技能       │
│                                                                 │
└────────────────────────────────────────────────────────────────┘

优势说明

和 JCo-MVTON 的核心区别

我之前用 JCo-MVTON（基于 FLUX）做过 VTON 研究，两者的根本差异在于：

JCo-MVTON 的思路：
  "我专门设计一个试衣架构，里面有 Reference-Net、Garment Encoder、
   DensePose 条件等，专门解决试衣问题"
  → 专家系统，试衣效果好但只能做试衣

Qwen-Image-Edit + LoRA 的思路：
  "我有一个超强的通用编辑模型，我用 20 张数据教它学会试衣这个技能"
  → 通才 + 技能包，灵活但需要精心调教

我的判断：对于商业项目，LoRA 方案的可维护性远高于专用架构。模型升级时只需重训 LoRA，不需要重写整个 pipeline。而且 Qwen-Image-Edit 在持续更新，底座能力的提升是"免费"的。

我的选择：Edit-2511

理由如下：

  1. 多图输入是刚需：VTON 至少需要两张输入图（人 + 衣服），Edit 原版只支持单图，必须放弃
  2. 2511 比 2509 质量更好：一致性更强、伪影更少，这对试衣效果至关重要
  3. 工具链最成熟：Musubi Tuner 和 AI-Toolkit 都有 edit-2511 的专属支持
  4. 2.0 虽然参数少但工具链尚未稳定：Qwen-Image 2.0 只有 7B 参数，效率诱人，但截至 2026 年初 LoRA 训练支持仍在完善中

⚠️ 关于 Qwen-Image 2.0：7B 参数 + Qwen3-VL 编码器 + 原生 2K 分辨率，理论上是更好的底座。但 Musubi Tuner 对 2.0 的 model_version 支持还在 PR 阶段（截至本文撰写时）。等工具链稳定后，2.0 很可能成为更优选择——参数少意味着训练更快、推理更便宜。

为什么是两阶段而不是一阶段？

你可能会问：为什么不直接训一个 LoRA 完成"模特 A 的衣服 → 穿到模特 B 身上"？

一阶段方案的问题：
  输入：模特A（穿着衣服A）+ 模特B
  期望：模特B 穿着衣服A
  
  问题：模型需要同时理解 —
  1. 从模特A身上"脱下"衣服（分离人体和衣物）
  2. 理解衣服的版型、颜色、细节
  3. 将衣服"变形"适配模特B的体型
  4. 保持模特B的身份特征不变
  
  → 一个 LoRA 承载太多任务，效果不稳定

两阶段方案的好处：
  Stage 1：只负责"脱衣服"（降低复杂度，任务单一）
  Stage 2：只负责"穿衣服"（输入已经是干净的衣服图）
  
  → 每个 LoRA 只做一件事，效果更好，也更容易调试

数据集准备（最关键的一步）

⚡ 这是整个项目成败的关键。 社区里大量"训不出好 LoRA"的帖子，99% 的原因都在数据集上。

理解 Edit 模型的数据三元组

Qwen-Image-Edit 的 LoRA 训练需要三元组——这一点和普通的 text-to-image LoRA 训练完全不同：

普通 LoRA 训练（如 FLUX、SD）：
  ┌──────────┐   ┌────────────┐
  │ 图片     │ + │ 文字描述   │  = 一个训练样本
  └──────────┘   └────────────┘

Edit 模型 LoRA 训练：
  ┌──────────┐   ┌────────────┐   ┌──────────┐
  │ 控制图片 │ + │ 编辑指令   │ + │ 目标图片 │  = 一个训练样本
  │ (输入)   │   │ (文字)     │   │ (输出)   │
  └──────────┘   └────────────┘   └──────────┘
  
  模型学的是：给定 [控制图片] + [编辑指令] → 生成 [目标图片]

Musubi Tuner 的 dataset.toml 配置

Musubi Tuner 使用 TOML 文件定义数据集。对于 Edit 模型的训练，关键是要正确指定 control_content：

# dataset_extract.toml — 衣服提取器 LoRA 数据集配置

[general]
resolution = [1024, 1024]
batch_size = 1
enable_bucket = true

[[datasets]]
image_directory = "path/to/dataset_extract/targets"
control_directory = "path/to/dataset_extract/controls"
caption_extension = ".txt"

# dataset_tryon.toml — 试穿器 LoRA 数据集配置
# 注意：2509/2511 支持多控制图

[general]
resolution = [1024, 1024]
batch_size = 1
enable_bucket = true

[[datasets]]
image_directory = "path/to/dataset_tryon/targets"
control_directory = "path/to/dataset_tryon/controls"
caption_extension = ".txt"
# 对于多控制图（人+衣服），需要在控制目录中正确组织

⚠️ 重要提示：Musubi Tuner 的 Edit 数据集配置要求控制图和目标图通过文件名配对。确保 control/001.jpg 对应 target/001.jpg，caption 文件为 001.txt。

VRAM 优化技巧

如果你手头只有 24GB 显卡（比如 RTX 4090），以下优化可以帮你跑通训练：

# Block swap：将部分 Transformer 块交换到 CPU 内存
--blocks_to_swap 20

# FP8 量化 Text Encoder
--fp8_vl

常见训练问题与解决

问题可能原因解决方案
Loss 不下降学习率太小 / 数据格式错误检查数据三元组是否正确配对
过拟合（训练好测试差）数据太少 / rank 太高降低 network_dim，增加数据量
OOM (显存不足)分辨率太高 / batch 太大增加 blocks_to_swap，降分辨率
输出有伪影训练步数不够 / 数据质量差检查数据质量，适当增加步数
衣服细节丢失LoRA 容量不足增加 network_dim 到 64
人体变形数据多样性不足增加不同体型的训练样本

完整推理代码

import torch
from PIL import Image
from diffusers import QwenImageEditPlusPipeline

# 加载基座模型
pipeline = QwenImageEditPlusPipeline.from_pretrained(
    "Qwen/Qwen-Image-Edit-2511",
    torch_dtype=torch.bfloat16
).to("cuda")

# ===== Stage 1: 衣服提取 =====

# 加载 Extract LoRA
pipeline.load_lora_weights(
    "path/to/extract_lora",
    weight_name="qwen_edit_extract_clothing.safetensors",
    adapter_name="extract"
)

# 输入模特照片
model_photo = Image.open("model_wearing_dress.jpg").convert("RGB")

# 提取衣服
extracted = pipeline(
    image=[model_photo],
    prompt="remove the person from this image, but leave the outfit. "
           "the clothes should remain after deleting the person's body, "
           "skin, and hair. leave the clothes in front of a white background",
    num_inference_steps=50,
    generator=torch.manual_seed(42)
).images[0]

# 卸载 Extract LoRA
pipeline.unload_lora_weights()

extracted.save("extracted_clothing.jpg")


# ===== Stage 2: 虚拟试穿 =====

# 加载 TryOn LoRA
pipeline.load_lora_weights(
    "path/to/tryon_lora",
    weight_name="qwen_edit_tryon_clothes.safetensors",
    adapter_name="tryon"
)

# 输入目标人物 + 提取出的衣服
target_person = Image.open("target_person.jpg").convert("RGB")

# 试穿！
result = pipeline(
    image=[target_person, extracted],
    prompt="dress the clothing onto the person",
    num_inference_steps=50,
    generator=torch.manual_seed(42)
).images[0]

# 卸载 TryOn LoRA
pipeline.unload_lora_weights()

result.save("tryon_result.jpg")
print("虚拟试穿完成！")

迭代优化路径

Round 1: 基线验证
  ├── 20 组数据 + 3000 步训练
  ├── 验证 Pipeline 是否跑通
  └── 收集第一批结果，找出主要问题

Round 2: 数据优化
  ├── 根据 Round 1 的问题，针对性补充数据
  ├── 比如衣服细节丢失 → 增加特写数据
  └── 比如体型变形 → 增加体型多样性

Round 3: 参数调优
  ├── 尝试不同 rank (16/32/64)
  ├── 尝试不同学习率 (5e-5 ~ 2e-4)
  └── 调整训练步数 (2000 ~ 5000)

Round 4: 质量提升
  ├── 增加数据到 50-100 组
  ├── 引入后处理（SeedVR2 超分、面部修复）
  └── 多种衣物类型测试（上衣/裤子/裙子/外套）

写在最后

回过头看，从 JCo-MVTON 到 Qwen-Image-Edit + LoRA 的方案迁移，本质上是一次范式转换：

旧范式：设计专用架构解决专用问题
  → 架构复杂、维护困难、无法复用

新范式：通用大模型 + 轻量微调 = 解决特定问题
  → 架构简洁、维护容易、底座升级免费享受

Qwen-Image-Edit 的 LoRA VTON 方案目前已经被社区验证可行（James Skelton 的教程证明 20 张图 + 3000 步就能用），而且随着 Qwen-Image 2.0（7B 参数、2K 分辨率）的成熟，未来只会更好。

最让我兴奋的一点是：这不只是换装。同样的思路可以扩展到发型试戴、配饰试戴、妆容试妆……只要你能构造出合适的训练三元组，LoRA 就能教模型学会新技能。
