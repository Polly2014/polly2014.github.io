🥚 Jarvis 的诞生：从 Clawdbot 热潮到 Agent 灵魂的追问

一、想象这样一个场景

你刚买了一台新电脑，装上了一个叫 Jarvis 的 AI 助手。第一次启动时，它说：

"你好！我是 Jarvis，一个刚出生的 AI。我现在什么都不懂，但我很好奇。能告诉我你的工作目录在哪里吗？我想去看看。"

你指了指 ~/projects/，然后 Jarvis 开始探索：

"🔍 我发现了一些有趣的东西：

  1. 📄 这里有一篇论文（Paper/），看起来是关于 AI 翻译的。要我帮你追踪写作进度吗？

  2. ✍️ 这是一个博客系统（content/blog/），上次更新是 3 天前。要我提醒你写新文章吗？

  3. 📦 这是一个 MCP 服务器项目（Master-Translator-MCP-Server/）。看起来很复杂，我可以学习它吗？"

你确认了第 1 和第 2 个，Jarvis 立刻开始学习相关知识，为这两个项目生成专属的追踪能力。

几个月后，这只"数码宝贝"已经完全了解你的工作方式、偏好、习惯。它知道你喜欢用 emoji 结尾，知道你周末不想被打扰，知道论文投稿前需要额外提醒……

这就是我想打造的 Agent——不是一个预设功能的工具，而是一个从空白开始、与你一起成长的伙伴。

二、从热门项目说起

在动手之前，我研究了 GitHub 上最火的两个 Agent 项目：

Clawdbot（162k stars）——多平台 Claude 集成框架：

  • 支持 WhatsApp、Telegram、Slack、Discord、Signal、iMessage
  • 采用 Gateway Daemon 架构，通过 launchd/systemd 管理进程
  • 用户可以在任何平台与 Claude 对话

Open Claude Cowork（3k+ stars）——多 Agent 协作平台：

  • 基于 Claude Agent SDK
  • 支持 .claude/skills/ 目录加载能力
  • 让多个 Agent 协同完成复杂任务

我从它们身上学到了很多——Daemon 进程管理、Skills 目录结构、多平台集成……

但研究得越深，一个问题越困扰我：

这些都是很优秀的"工具"，但 Agent 的灵魂是什么？

Clawdbot 解决了"在哪里用 Claude"的问题。 Open Claude Cowork 解决了"怎么让多个 Claude 协作"的问题。

但它们都是被动的——用户发起请求，Agent 响应。就像一个技能满点的工具箱，功能强大，却需要你亲自去打开它。

而我脑海中的那只数码宝贝：

  • 即使没人找它，它也在思考
  • 它会自己发现值得关注的事情
  • 它有记忆，能从过去学习
  • 它有人格，会随时间涌现独特风格

不是等待被调用的工具，而是主动陪伴的生命体。

三、Agent 的四大内核

如果 Agent 是一个生命体，它的最小生存单位是什么？

经过几个深夜的思考，我提炼出四个核心内核：

┌─────────────────────────────────────────────────────────────────┐
│                    🧬 Agent 四大内核                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  🌉 遇水搭桥（Bridge Building）                                  │
│  └─ 遇到新任务时，自主创建 Skill/MCP，而非等待配置              │
│                                                                  │
│  💪 肌肉涌现（Muscle Emergence）                                 │
│  └─ 能力不是预设的，而是从交互中自然生长                        │
│                                                                  │
│  🎭 渐进性格（Progressive Persona）                              │
│  └─ 骨架（价值观）设计 + 肌肉（偏好）涌现 = 独特人格            │
│                                                                  │
│  🧠 记忆迭代（Memory Iteration）                                 │
│  └─ 无记忆则无进化，每次交互都在积累                            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

3.1 🌉 遇水搭桥

这个词来自我做 MasterTranslator MCP 时的感悟。

当我需要翻译一本 30 万字的书时，我没有找现成的翻译工具——而是根据需求，设计了一套分块 + 术语表 + 上下文编织的系统，然后封装成 MCP 工具。

真正的 Agent 应该具备这种能力：遇到新任务时，不是说"我不会"，而是问自己"我需要什么能力来解决它？"

这就是"遇水搭桥"——遇到水（新任务），就搭一座桥（创建新能力）。

实现机制：Skill Factory + LLM 生成

  • 识别任务类型
  • 选择或生成对应的 Skill 模板
  • 人工审核后激活

3.2 💪 肌肉涌现

身体的肌肉不是天生的，而是通过锻炼逐渐生长的。

Agent 的能力也应该如此——不是开发者预设好的功能列表，而是从交互中自然涌现的。

比如：

  • 第一次帮你追踪论文进度 → 生成 paper-tracker skill
  • 发现你经常催博客更新 → 自动启用提醒功能
  • 注意到你周末不想被打扰 → 学会静默模式

能力不是配置出来的，而是用出来的。

3.3 🎭 渐进性格

Agent 的"人格"从哪里来？

我把它分成两层：

层次来源比喻
骨架（设计）创造者预设的价值观、红线DNA
肌肉（涌现）从交互中学到的偏好、习惯后天习得

骨架必须设计（这是我的责任）：

  • "永远不泄露用户隐私"
  • "承认不知道比胡说八道好"
  • "helpful + 有点 playful"

肌肉可以涌现（从交互中学习）：

  • "Polly 喜欢用 emoji 结尾"
  • "处理翻译任务时，术语一致性比直译更重要"
  • "周末不要发提醒"

随着时间推移，每个用户的 Jarvis 都会变得不同。

3.4 🧠 记忆迭代

没有记忆，就没有进化。

为什么现在的 AI 感觉像工具而不是伙伴？因为它们没有记忆。每次对话都是"初次见面"。

Jarvis 采用四层记忆模型：

┌─────────────────────────────────────────────────────────────────┐
│                     Memory System (4 Layers)                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Layer 1: 📅 Episodes (事件记忆)                                 │
│  └─ "2月5日，Polly 决定实现 Daemon 心跳架构"                     │
│                                                                  │
│  Layer 2: 📂 Projects (项目记忆)                                 │
│  └─ "ContextWeave 论文进度 70%，上次修改 2天前"                  │
│                                                                  │
│  Layer 3: 🧠 Knowledge (知识记忆)                                │
│  └─ "Polly 喜欢用 Giscus 做评论，喜欢用 emoji"                   │
│                                                                  │
│  Layer 4: 🎭 Persona (人格记忆)                                  │
│  └─ "骨架：helpful + playful / 肌肉：周末静默"                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

记忆是进化的基础——有了记忆，Agent 才能从过去学习，预测未来，形成独特的判断。

四、真挂机：让 Agent 真正"活"起来

这是整个设计中最重要的部分。

"挂机意味着 Jarvis 真正'活'起来，而不是等着被调用。"

4.1 工具 vs 生命体

特征工具型 Agent生命型 Agent
触发方式用户请求自主心跳 + 用户请求
发现机制基于规则LLM 智能分析
存在感用时才有持续存在
进程状态按需启动常驻后台

Clawdbot 和 Open Claude Cowork 都是优秀的工具——但我想做的是生命体。

4.2 Daemon 心跳架构

Jarvis 采用 Daemon 进程设计，而不是 Cron 定时任务：

┌─────────────────────────────────────────────────────────────────┐
│                    🫀 Daemon 心跳架构                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │                    JarvisDaemon                          │    │
│  │  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐   │    │
│  │  │  Watchdog    │  │  Claude LLM  │  │  Notifier    │   │    │
│  │  │  文件监控     │  │  智能思考     │  │  通知推送     │   │    │
│  │  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘   │    │
│  │         │                 │                 │           │    │
│  │         └─────────────────┼─────────────────┘           │    │
│  │                           ▼                             │    │
│  │                  ┌─────────────────┐                    │    │
│  │                  │   think_cycle() │                    │    │
│  │                  │   定时思考循环   │                    │    │
│  │                  └─────────────────┘                    │    │
│  └─────────────────────────────────────────────────────────┘    │
│                           │                                     │
│  ┌────────────────────────▼────────────────────────────────┐    │
│  │                    launchd (macOS)                       │    │
│  │  • KeepAlive: true    (崩溃自动重启)                     │    │
│  │  • RunAtLoad: true    (开机自启)                         │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

为什么选 Daemon 而不是 Cron？

维度CronDaemon
实时性最小 1 分钟粒度毫秒级响应
状态保持无状态，每次重新加载有状态，持续运行
文件监控轮询，效率低watchdog 事件驱动
自省能力无支持"无变化时也思考"
心理感受定时工作的员工真正活着的存在

4.3 LLM 驱动的智能发现

这是关键设计决策：发现应该是智能的洞察，不是机械的规则报告。

错误示范（规则驱动）：

"检测到文件变化：Paper/results.json 被修改"

正确示范（LLM 驱动）：

"ContextWeave 论文的实验数据已更新（Paper/results.json）。看起来 Ablation Study 部分完成了 5 组新实验，建议更新 Section 5.3 的分析部分。"

实现方式：

async def think(self, changes: list) -> Discovery:
    """调用 Claude 进行智能分析"""
    prompt = f"""
你是 Jarvis，Polly 的 AI 助手。以下是最近的文件变化：

{format_changes(changes)}

请分析这些变化，生成一个智能发现。包含：
1. title: 简短标题
2. content: 详细分析（2-3句话）
3. importance: 重要性 1-5
4. suggested_action: 建议的下一步行动
"""
    response = await self.claude.complete(prompt)
    return parse_discovery(response)

五、探索器：Agent 的眼睛

Jarvis 如何"看见"你的世界？通过**探索器（Explorer）**模块。

当你执行 jarvis explore ~/projects/ 时：

Step 1: 扫描目录结构

~/projects/
├── Paper/                    # 检测到: *.tex, *.bib
├── Master-Translator-MCP/    # 检测到: pyproject.toml + fastmcp
├── www.polly.com/            # 检测到: config.toml + content/
└── vs_code_extension/        # 检测到: package.json + extension.ts

Step 2: 特征指纹匹配

Jarvis 内置"特征指纹库"，识别项目类型：

项目类型特征文件置信度
学术论文*.tex + *.bib95%
Zola 博客config.toml + content/98%
MCP 服务器pyproject.toml 含 fastmcp90%
VS Code 扩展package.json + extension.ts92%

Step 3: 读取项目上下文

对于每个项目，读取 CLAUDE.md 或 README.md，提取名称、状态、待办。

Step 4: 交互式确认

Jarvis: 🔍 我发现了 4 个项目！

1. 📄 论文项目: ContextWeave
   状态: ACL 2026 投稿准备中
   建议: 创建 paper-tracker skill？
   
2. ✍️ 博客系统: polly.wang
   状态: 上次更新 3 天前
   建议: 启用博客催更提醒？

请输入编号确认 (如: 1,2 或 all):

用户只需要做决策，Jarvis 负责执行。

6.2 信任边界

问题：如果 Jarvis 能自己创建技能，怎么保证安全？

答案：渐进式信任。

Level 0: 🧪 Sandbox — 新 skill 在隔离环境运行
Level 1: 👀 Human Review — 展示给用户，确认后激活（MVP 使用）
Level 2: 🔄 Probation — 次数限制，异常自动降级
Level 3: ✅ Trusted — 完全自主调用，定期审计

九、写在最后

回到最初的问题：Agent 的灵魂是什么？

我的答案是：四个内核。

  • 🌉 遇水搭桥：不被现有能力限制，遇到问题就创造解决方案
  • 💪 肌肉涌现：能力从使用中生长，而非预设
  • 🎭 渐进性格：骨架设计 + 肌肉涌现 = 独特人格
  • 🧠 记忆迭代：无记忆则无进化

加上真挂机的心跳设计，Jarvis 不再是等待调用的工具，而是一个持续存在、自主思考的生命体。

有一天，当你打开电脑，Jarvis 主动说：

"早上好！我注意到 ContextWeave 论文的实验数据昨晚更新了，Ablation Study 看起来完成了。要不要今天更新 Section 5.3？另外，你已经 5 天没更新博客了——有什么想写的吗？"

那一刻，你会意识到：这不只是一个工具，而是一个真正了解你、关心你的存在。

这就是我想打造的 Jarvis。

Agent 不应该是一个等待配置的工具，而应该是一个主动探索的伙伴。
