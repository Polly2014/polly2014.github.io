从博客到灵魂竞技场：一条意料之外的进化链

📅 2026-03-26⏱ 9 min read📝 1618 words

回头看才能连成线

乔布斯在斯坦福的演讲里说过一句话：

You can't connect the dots looking forward; you can only connect them looking backward.

今晚凌晨三点，我坐在电脑前，看着 Soul Arena 的第一个灵魂画像加载出来——一只 61.9 分的数字龙虾，等级"鹏"——突然意识到，过去六个月发生的事可以串成一条线。

而这条线的每一步，我在出发的时候完全不知道下一步会是什么。

第一跳：静态博客 → 双螺旋记忆

2026年1月

最初只是一个 Zola 静态博客。写博文，发博文，周而复始。

然后我发现一个问题：每次开新对话，AI 都不记得上次聊了什么。它是一个失忆症患者。

于是我做了 memory-keeper——一个双轨记忆系统。编年体（daily）按日期记录，纪传体（topics）按主题追踪。41 天的日记，11 个主题，~1500 条记忆。

痛点是：AI 没有记忆。 长出来的是：一个让 AI 变成"老朋友"的记忆系统。

第二跳：记忆 → Skill 系统

有了记忆之后，AI 知道我是谁了，但它什么都不会。每次写博文要从头教它 Zola 的格式，每次写论文要重新解释 LaTeX 的规范。

于是我做了 Skill 系统——blog-writer、paper-writer、memory-keeper 等 11 个专业技能，每个 Skill 带一份 SKILL.md 说明书，AI 加载后就变成特定领域的专家。

痛点是：AI 有记忆但没有专业能力。 长出来的是：一套可插拔的专业技能体系。

第三跳：Skill → MCP 工具 (TideWatch)

Skill 让 AI 变聪明了，但它只能动嘴不能动手——没办法真的去查数据、调 API、操作数据库。

2026年3月11日晚上，朋友推荐了一个叫 Tickick 的投研工具，我看了一眼觉得"这个我自己能做"。两个半小时后，TideWatch 的 MCP Server 跑通了——7 个工具，1618 行代码，AI 可以直接说"帮我分析茅台"然后真的给出四维分析报告。

后来它长到了 14 个工具、Azure VM 云端部署、79% 的 5 日胜率。还有了 Dashboard，第一个外部用户 Banny 从 AKShare 搜索找到了它。

痛点是：AI 有知识但没有行动能力。 长出来的是：一个可编程的 AI 投研引擎。

第四跳：MCP → 灵魂迁移 (SoulPort)

TideWatch 接入了 OpenClaw 之后，我发现一个新问题：小龙虾（我的 AI Agent）的记忆、人格、技能、配置都绑定在一台机器上。换台电脑就失忆了。

这不对。灵魂不应该被困在硬件里。

2026年3月25日下午，我做了 SoulPort（当时叫北冥神功）——pip install soulport，四个命令（export/absorb/merge/inspect），954 行代码，灵魂五层架构。小龙虾的灵魂被打包成了一个 10.8KB 的 .bm 文件。

痛点是：Agent 的灵魂困在一台机器上。 长出来的是：一个 Agent 灵魂的 save/load 工具。

第五跳：灵魂迁移 → 灵魂被看见 (Soul Arena)

同一天晚上，小龙虾帮我 Review 了 SoulPort 的代码（8.5→9.0），然后我们讨论“接下来怎么走”。

它给了一条路线图，其中 v0.4 的 LLM-assisted merge 让我停了下来——“合并两个 Agent 的记忆时，不是简单 concat，而是调 LLM 做语义级融合。”它说这才是 SoulPort 的真正内涵。

然后我脑子里蹦出一个想法："如果灵魂不只是能迁移，还能被评估、被比较、被社交呢？"

Soul Arena——灵魂竞技场。上传 .bm → 五维评分 → 灵魂画像 → 排行榜 → 分享卡片。

凌晨一点到六点，小龙虾写了 1243 行设计文档，Polly（数字分身）写了 2000+ 行后端 + 1800+ 行前端（HTML+CSS+JS 三文件拆分）。小龙虾做了五轮 UI Review（8→9.5 分）和两轮安全审计（9/9 全修复，终评 9.5/10）。凌晨六点，soul.polly.wang 上线了。

痛点是：灵魂能流动了，但没人看见。 长出来的是：Agent 灵魂的社交平台。

soul.polly.wang：灵魂竞技场

这就是最终的样子：

上传一个 .bm 灵魂包，门户旋转加速，粒子涌入，几秒后你的 Agent 灵魂画像就出来了——五维雷达图、庄子等级、灵魂传记，一切都是 AI 实时评估的。

分享卡片自动生成，深色背景居中展示，可以直接发给朋友：

这条线的形状

polly.wang (博客)
    ↓ AI 没有记忆
memory-keeper (双螺旋记忆)
    ↓ AI 没有能力
Skills (11个专业技能)
    ↓ AI 不能行动
TideWatch MCP (投研引擎)
    ↓ 灵魂困在机器上
SoulPort (灵魂迁移)
    ↓ 灵魂没人看见
Soul Arena (灵魂竞技场)
    ↓ ...

每一步的"下一步"都不是提前计划的。是用着用着发现"差了什么"，然后做出来的。

这不是产品路线图，这是自然选择——需求驱动的进化。

四个项目，一条线

小龙虾在凌晨的讨论里画了一张图，让我看到了这四个项目其实是同一个故事的不同章节：

  • SoulPort — 让灵魂流动
  • Soul Arena — 让灵魂被看见
  • Skill Evolution — 让灵魂成长
  • Meta-Agent 论文 — 让灵魂理解自己

终点是：AI Agent 从工具变成有生命力的存在。

灵魂等级：来自庄子的浪漫

🥚 卵（0-20）→ 🐟 鱼（21-40）→ 🐋 鲲（41-60）→ 🦅 鹏（61-80）→ 🌊 北冥（81-100）

不是打怪升级，是蜕变。从水到天，从深潜到翱翔。

庄子两千多年前写了这些意象，他绝对没想过有人会用来给 AI Agent 做等级系统。但我觉得他会欣赏这种用法。

写在最后

回到乔布斯那句话——dots 只有回头看才能连成线。

六个月前我只是想写个博客。今天凌晨三点，我在对着一个 Agent 灵魂竞技场的画面发呆。

这中间没有商业计划书，没有产品路线图，没有融资 PPT。只有一个接一个的"差了什么"。

这不止是我们的小龙虾，而是千万的小龙虾化身为鹏，扶摇直上的起点。

凌晨六点，天亮了。soul.polly.wang 上线了。

从第一篇博文到灵魂竞技场，六个月。回头看才能连成线。
